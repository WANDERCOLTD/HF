# First Course — Demo Scenarios & Priorities

## Goal

Harden the end-to-end flow for our first real course. This serves two purposes:

1. **System test** — validate the full loop works (create course → upload content → onboard student → call → adaptive response)
2. **Investor demo** — a polished walkthrough that shows the product's value

## How to contribute

1. Open your file (`boaz-investor.md` or `paul-technical.md`)
2. Add your scenarios using the template below
3. Commit + push — Claude will groom top items into GitHub Issues

### Scenario template

```markdown
## Scenario: [Name]

**Purpose:** Why this matters for the demo / investor pitch
**Screens:** List the screens/pages involved
**Happy path:** Step-by-step what the viewer should see
**Must work:** Non-negotiable things that can't break
**Nice to have:** Polish items if time allows
```
