## How to ingest in `/x/wizard`

1. Open `/x/wizard` on hf-dev VM (or whichever environment).
2. Paste the **chat opener** below into the wizard chat.
3. Wait for the wizard to confirm subject + audience + goal.
4. When the wizard surfaces the Teaching Materials panel on the right, drop all five files above into it (in any order — the wizard classifies by front-matter).
5. Wait for upload + classification confirmation in the Sources panel. Watch for any `sourceWarnings` — those mean a front-matter value was rejected and the AI fell back to inference.
6. Continue the chat to confirm the proposal playback.
7. Once "Can launch: YES" + all preconditions are met (see the `create_course` gate in `lib/chat/v5-system-prompt.ts`), click "Ready to create my course".

### Chat opener (paste verbatim)

```
I want to build a 5-call demo course on the Big Five (OCEAN) personality model
for general adult learners who've never used AI tutoring before. Voice-only,
about 7 minutes per call. Goal is for them to walk away able to name the five
traits, honestly self-locate on each, and catch the common folk misuses like
"introvert = shy" or "neurotic = mentally ill". Socratic style, never lectured at.
```

## Post-create admin steps (set in `/x/courses/[id]/config`, not in the wizard)

These fields are **not** Wizard-settable. After the wizard creates the course, edit them via the course config page (or directly via Prisma Studio if needed):

1. **`Playbook.config.skillScoringEmaHalfLifeDays = 4`** — tightens the mastery-graph response so the dashboard shows visible movement in a 5-call demo (default is 14, which is too slow at this scale).
2. **Module-specific deliveryHints overrides** (optional) — if you want the Reading Profiles + Cross-cultural module to feel more reflective than the trait modules, the per-module `teachingOverrides.deliveryHints` can be set in the same config page.

## What the wizard will do with these files

| File | What gets created in the DB |
|---|---|
| Course Ref → projection | `CurriculumModule` (6 rows), `Goal type:LEARN` (16 rows for OUT-NN), `Goal type:ACHIEVE` + `Parameter` + `BehaviorTarget` per SKILL-NN (3 sets), `ContentAssertion` rows tagged INSTRUCTION_CATEGORIES from Teaching Approach + Edge Cases. `Playbook.config.{modules, outcomes, modulesAuthored, moduleSource, progressionMode}` written. |
| TEXTBOOK × 2 | `ContentAssertion` rows (concept / vocabulary / key_term) via the chunked LLM extractor pyramid. |
| QUESTION_BANK | `ContentQuestion` rows (one per item) tagged `assessmentUse: FORMATIVE`, `bloomLevel: APPLY`, with model responses + misconception traps. |
| REFERENCE | `ContentAssertion` rows tagged `definition` / `vocabulary` for each glossary entry. |

After projection runs, the Playbook is ready to take a real call.

## Source provenance

All sources cited in the Course Reference are `ContentTrustLevel: PUBLISHED_REFERENCE` (L3) at minimum. No `UNVERIFIED` sources. The MBTI / 16Personalities folk taxonomies are explicitly named in the Course Reference as misconceptions to catch, not as authoritative sources.
